# Pancakes: A View From Space

Pancakes is an open ecosystem for supporting human flourishing.

It is built on the idea that software should help people care for themselves,
their households, their communities, and the places they live without requiring
them to surrender ownership of their lives or their data.

Unlike a single application or online platform, Pancakes is an ecosystem of
locally governed software built around shared principles and shared
infrastructure.

Its long-term goal is to make cooperation easier while making surveillance,
vendor lock-in, and extractive business models less necessary.

---

## The Core Architecture

The ecosystem is built from several complementary layers.

At the center is **Pitchfork**.

Pitchfork is the shared symbolic and accounting substrate used by every major
Pancakes application.

It records permissioned events, settles contracts, manages symbolic resources,
and produces projections that applications can interpret in different ways.

Around Pitchfork are many independent applications.

Applications own user experience.

Pitchfork owns accounting.

Nodes own governance.

People own their lives.

---

## Nodes Rather Than Platforms

Pancakes is designed around locally governed nodes rather than one centralized
service.

A node may represent:

* one person;
* one household;
* one cooperative;
* one guild;
* one school;
* one clinic;
* or an official hosted deployment.

Nodes control their own identities, permissions, policies, exports, and
governance.

Official hosted services are intended to be convenient deployments of the same
architecture, not the architecture itself.

---

## Flagship Projects

The ecosystem contains several major projects, each validating a different part
of the overall design.

### Pitchfork

Pitchfork is the architectural core of the ecosystem.

Every major application either produces events for Pitchfork, consumes
Pitchfork projections, or both.

It provides the common symbolic language that allows many applications to share
one coherent model without becoming one monolithic program.

### Service Exchange

Service Exchange is the flagship Pancakes application.

It explores how households, communities, cooperatives, and local economies can
coordinate services, stewardship, contracts, and mutual aid using the shared
Pancakes infrastructure.

It demonstrates the broader social vision of the ecosystem.

### Red Witch

Red Witch demonstrates the ecosystem's privacy model.

It applies the same architecture to one of the most privacy-sensitive domains:
menstrual and reproductive health.

If the architecture can responsibly support Red Witch, it provides evidence
that the broader ecosystem can be trusted with other forms of intimate personal
data.

### Other Clients

Other applications—including wellness tools, educational software, symbolic
games, environmental observation, and future community services—share the same
underlying architecture while presenting very different experiences.

---

## A Different Relationship Between Applications

Most software products create isolated silos.

Each application maintains its own accounts, permissions, activity history, and
interpretation of user behavior.

Pancakes separates these responsibilities.

Applications provide experiences.

Pitchfork provides shared accounting.

Nodes provide governance.

Communities provide meaning.

This allows multiple applications to cooperate without forcing users into a
single platform.

---

## Human-Centered Infrastructure

The ecosystem is built around several recurring principles.

* People should own their own records.
* Communities should be able to govern themselves.
* Local deployment should remain practical.
* Privacy should be the default.
* Export should be a first-class capability.
* Participation should be recognized without becoming exploitative.
* Technology should strengthen stewardship rather than replace it.

These principles apply equally to technical architecture and social
organization.

---

## Long-Term Direction

The long-term ambition of Pancakes is not to become another social network,
marketplace, or cloud platform.

Instead, it aims to provide a shared foundation upon which many independent
applications and communities can build.

As the ecosystem grows, independent nodes may exchange services, knowledge,
computation, and resources while remaining locally governed and interoperable.

The result is intended to be an ecosystem rather than a platform: many humane
applications sharing one coherent foundation instead of one application trying
to become everything.

---

## In One Sentence

Pancakes is an open, locally governed ecosystem for human flourishing, built on
a shared symbolic infrastructure that allows many applications and communities
to cooperate without surrendering autonomy.
