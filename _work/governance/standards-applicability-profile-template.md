# Purpose

The Standards Applicability Profile (SAP) is the primary governance intake document for the Pancakes ecosystem.

Its purpose is to describe:

* what the project is,
* what the project does,
* who uses it,
* what data it handles,
* how it operates,
* how value is exchanged,
* what risks are present,
* and what governance obligations arise.

The SAP does not determine whether a project is acceptable.

Instead, the SAP provides the information necessary to determine:

* applicable standards,
* applicable regulations,
* applicable safety cases,
* applicable ethics cases,
* applicable governance reviews.

The SAP is the primary input to governance review.

# Change Management

The SAP shall be reviewed whenever:

* intended use changes,
* user populations change,
* AI capabilities change,
* economic mechanisms change,
* sensitive data classes change,
* deployment architecture changes,
* new hazards emerge,
* new ethical concerns emerge.

Changes that materially affect classification may require:

* standards reassessment,
* safety case reassessment,
* ethics reassessment,
* governance reapproval.
